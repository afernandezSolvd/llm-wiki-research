# Implementation Plan: Obsidian Read-Only Vault via Git Remote Sync

**Branch**: `004-obsidian-git-sync` | **Date**: 2026-04-24 | **Spec**: [spec.md](spec.md)

## Summary

Push each workspace's local git repo to a remote (GitHub or GitLab) after every wiki write, so developers can clone the remote and view an always-current wiki in Obsidian. Three additions: (1) a `git_push` Celery worker that serializes pushes per workspace using a Redis lock, (2) three new nullable columns on `workspaces` to track remote URL and push state, and (3) a `GET /clone-url` endpoint that returns the HTTPS clone URL to workspace members.

## Technical Context

**Language/Version**: Python 3.12 (existing)  
**Primary Dependencies**: gitpython (existing), httpx or urllib.request for provider API calls (no new SDK), redis-py (existing, for push serialization lock)  
**Storage**: PostgreSQL 16 — three new nullable columns on `workspaces` table; Redis — distributed lock key per workspace  
**Testing**: pytest + pytest-asyncio (existing)  
**Target Platform**: EKS (Linux) — provider token injected via Kubernetes Secret → env var  
**Project Type**: Extension to existing web-service + async worker system  
**Performance Goals**: Push completes within 60 seconds of a wiki commit; push failure adds zero latency to ingest  
**Constraints**: Push MUST be non-blocking (async Celery task); only one push per workspace at a time (Redis lock); feature fully disabled when `WIKI_GIT_ENABLED=false`  
**Scale/Scope**: One remote repo per workspace; one git provider per deployment
